Sobre el Proyecto

Este proyecto fue el resultado de mi formación en el Bootcamp Full Stack de Henry, donde desarrollé de forma individual una API REST completa para un sistema de e-commerce utilizando tecnologías modernas del ecosistema Node.js.

Aunque fue un proyecto académico guiado, cada estudiante lo construyó de manera independiente, tomando decisiones propias sobre la arquitectura, implementación de features y manejo de la lógica de negocio. El instructor nos mostraba un proyecto similar, pero cada uno debía implementar su propia solución.

El proyecto culminó con una defensa técnica ante un profesional externo, donde debí explicar todas las decisiones de arquitectura, responder preguntas técnicas profundas sobre la implementación, y realizar modificaciones en vivo del código (en mi caso, modificar un endpoint bajo presión).

Contexto del Proyecto

Bootcamp: Henry Full Stack Development
Modalidad: Proyecto individual con guía del instructor
Evaluación: Defensa técnica ante profesional externo
Objetivo: Demostrar dominio de backend moderno con NestJS, TypeORM y PostgreSQL

¿Qué Construí?

Sistema de Autenticación

Implementé autenticación completa con JWT, incluyendo registro de usuarios, login, hash de contraseñas con bcrypt, y un sistema de roles (usuario regular vs administrador) con guards personalizados para proteger endpoints específicos.

Gestión de Productos

CRUD completo de productos con paginación, relaciones entre productos y categorías usando TypeORM, sistema de carga masiva desde JSON (seeder), y subida de imágenes a Cloudinary para no depender del almacenamiento local.

Sistema de Órdenes

Creación y gestión de órdenes de compra con relaciones complejas entre usuarios, órdenes, detalles de órdenes y productos. Implementé la lógica para calcular totales, gestionar stock y mantener la integridad de los datos.

Documentación Swagger

Configuré Swagger/OpenAPI para generar documentación interactiva automática de toda la API, incluyendo ejemplos de requests, responses esperadas, códigos de estado, y requisitos de autenticación para cada endpoint.

Stack Tecnológico Utilizado

Framework Backend

NestJS 10.0 TypeScript 5.1 Arquitectura Modular Decoradores

Utilicé NestJS por su arquitectura modular inspirada en Angular, que facilita la separación de responsabilidades y el testing. TypeScript me permitió tener tipado estático y detectar errores en tiempo de desarrollo.

Base de Datos

PostgreSQL TypeORM 0.3 Migraciones Relaciones

Diseñé el esquema de base de datos con relaciones 1:N y N:N entre entidades. TypeORM me permitió manejar estas relaciones de forma elegante y type-safe, además de gestionar migraciones de forma controlada.

Seguridad y Validación

JWT bcrypt class-validator Guards Personalizados

Implementé validaciones robustas con class-validator, creé guards personalizados para autenticación y autorización por roles, y aseguré las contraseñas con bcrypt. También validé formatos de email, longitudes de campos y coincidencia de contraseñas.

Herramientas Adicionales

Swagger/OpenAPI Cloudinary Docker Jest (Testing)

Documenté la API con Swagger, integré Cloudinary para gestión de imágenes, dockericé el proyecto para portabilidad, y escribí tests unitarios con Jest para asegurar la calidad del código.

Desafíos Técnicos y Soluciones

Relaciones Complejas en TypeORM

Al principio me costó entender cómo manejar correctamente las relaciones N:N entre productos y detalles de órdenes, y cómo hacer que TypeORM manejara automáticamente las tablas intermedias.

Lo que Aprendí

Estudié la documentación de TypeORM a fondo y experimenté con @ManyToMany, @JoinTable y opciones de eager loading. Aprendí a usar eager: true solo donde era necesario para evitar queries innecesarias, y cómo estructurar correctamente las entidades para que las relaciones fueran bidireccionales.

Validación de Contraseñas

Necesitaba validar que la contraseña cumpliera requisitos de seguridad (mayúsculas, minúsculas, números, símbolos) y además que coincidiera con el campo de confirmación.

Lo que Aprendí

Creé mi primer decorador personalizado @MatchPassword que compara dos campos del DTO. También implementé validaciones con regex para asegurar contraseñas fuertes. Fue mi primera vez creando decoradores propios y entendí el poder de los decoradores de NestJS.

Guards de Autorización

Necesitaba proteger ciertos endpoints para que solo los administradores pudieran acceder (como actualizar productos), pero sin duplicar código de validación en cada controlador.

Lo que Aprendí

Implementé un RolesGuard reutilizable que lee metadata de los decoradores @Roles() y valida si el usuario tiene el rol necesario. Combiné este guard con el AuthGuard de Passport para tener autenticación + autorización en un solo lugar limpio y mantenible.

Carga Masiva de Productos

Necesitaba poblar la base de datos con productos de prueba desde un archivo JSON sin crear duplicados cada vez que ejecutara el seeder.

Lo que Aprendí

Investigué sobre operaciones upsert (insert or update) en PostgreSQL y las implementé a través de TypeORM. Creé un endpoint /products/seeder que verifica si el producto ya existe antes de insertarlo, evitando duplicados y permitiendo re-ejecutar el seeder múltiples veces de forma segura.

Por Qué Este Proyecto Es Importante para Mí

Aunque fue un proyecto académico, representa un punto de inflexión en mi formación. Por primera vez construí una API completa con arquitectura profesional, no solo scripts sueltos. Aprendí a pensar en escalabilidad, seguridad y mantenibilidad desde el inicio.

La defensa técnica me enseñó algo crucial: no basta con hacer que el código funcione; hay que entenderlo profundamente para poder explicarlo y modificarlo bajo presión. Esa experiencia me preparó para entrevistas técnicas y proyectos reales donde necesito justificar decisiones de arquitectura.

Implementación Técnica Destacada

Arquitectura de Módulos

Estructura modular de NestJS implementada:

  • auth/ - Autenticación JWT, guards y estrategias de Passport
  • users/ - Gestión de usuarios, roles y perfiles
  • products/ - CRUD de productos, categorías y stock
  • orders/ - Sistema de órdenes y detalles de compra
  • file-upload/ - Integración con Cloudinary para imágenes
  • config/ - Configuración centralizada de TypeORM y variables de entorno

Modelo de Datos Implementado

  • Users: Entidad con validaciones, hash de contraseñas y relación con órdenes
  • Products: Catálogo con stock, precios, imágenes y relación con categorías
  • Categories: Clasificación de productos con relación 1:N
  • Orders: Órdenes de compra vinculadas a usuarios
  • OrderDetails: Tabla intermedia con productos y precios de cada orden
  • Relaciones N:N entre Products y OrderDetails manejadas automáticamente por TypeORM

Sistema de Seguridad

  • Contraseñas hasheadas con bcrypt (salt rounds configurables)
  • JWT con payload personalizado (id, email, isAdmin) y expiración de 1 hora
  • Guards personalizados: AuthGuard para autenticación, RolesGuard para autorización
  • Decorador @Roles() para proteger endpoints específicos
  • Validaciones con class-validator: emails únicos, contraseñas fuertes, campos requeridos
  • Decorador personalizado @MatchPassword para confirmar contraseñas

Features Implementadas

  • Paginación en endpoints de listado (GET /products?page=1&limit=5)
  • Seeder de productos desde archivo JSON con upsert para evitar duplicados
  • Subida de imágenes a Cloudinary con URLs persistentes
  • Middleware de logging global para tracking de requests
  • Documentación automática con Swagger/OpenAPI en /api
  • Variables de entorno con validación de configuración
