Sobre el Proyecto

"Si, Voy" es un portal innovador que promueve el turismo accesible en Argentina, conectando a personas con discapacidad con prestadores de servicios turísticos que ofrecen experiencias verdaderamente inclusivas.

La plataforma actúa como puente entre la comunidad turística y la discapacidad, democratizando el acceso al turismo y garantizando que todas las personas, independientemente de sus capacidades físicas, puedan disfrutar de experiencias de viaje memorables.

Durante mi participación en este proyecto, trabajé en la reestructuración completa del backend, implementando mejoras significativas en la arquitectura de la API, optimizando consultas a la base de datos y colaborando estrechamente con el equipo frontend para garantizar una integración fluida y eficiente.

Mi Rol en el Proyecto

Como Backend Developer, fui responsable de la reestructuración completa del sistema, trabajando en un equipo colaborativo para transformar la arquitectura existente en una solución más robusta, escalable y mantenible.

¿Qué Ofrece la Plataforma?

Experiencias de Viaje

Catálogo completo de prestadores turísticos accesibles en toda Argentina, incluyendo destinos de invierno como el Esquí Adaptado El Bolsón, hoteles accesibles, excursiones adaptadas y actividades recreativas diseñadas para personas con movilidad reducida.

Capacitaciones Empresariales

Cursos especializados para empresas del sector turístico sobre cómo atender adecuadamente a personas con discapacidad, promoviendo el diseño universal y el trato adecuado. Estas capacitaciones ayudan a transformar la industria turística hacia una cultura más inclusiva.

Accesibilidad Garantizada

Sistema de verificación y certificación de accesibilidad que garantiza que todos los servicios listados cumplan con estándares reales de inclusión, proporcionando información detallada sobre las adaptaciones disponibles en cada establecimiento.

Conexión Directa

Plataforma de contacto directo entre personas con discapacidad y prestadores de servicios, facilitando la comunicación de necesidades específicas y permitiendo una planificación personalizada de cada experiencia turística.

Problemáticas Resueltas

Arquitectura Legacy Poco Escalable

El sistema original presentaba una arquitectura monolítica con código altamente acoplado, dificultando la implementación de nuevas funcionalidades y el mantenimiento del código existente.

Solución Implementada

Reestructuración completa hacia una arquitectura modular basada en capas (controllers, services, repositories), aplicando principios SOLID y patrones de diseño que facilitaron la escalabilidad y el testing. Implementé separación de responsabilidades clara, permitiendo que cada módulo sea independiente y reutilizable.

Consultas SQL Ineficientes

Múltiples endpoints presentaban problemas de rendimiento debido a consultas N+1, falta de índices apropiados y queries mal optimizadas que generaban tiempos de respuesta elevados.

Solución Implementada

Optimización de consultas mediante eager loading, creación de índices estratégicos en PostgreSQL, y refactorización de queries complejas. Reduje el tiempo de respuesta promedio en un 60% para las operaciones más críticas, mejorando significativamente la experiencia del usuario.

Falta de Validación y Manejo de Errores

El sistema carecía de validación robusta de datos de entrada y manejo consistente de errores, lo que resultaba en respuestas poco claras y potenciales vulnerabilidades de seguridad.

Solución Implementada

Implementé middleware de validación centralizado utilizando librerías como class-validator, establecí un sistema consistente de manejo de errores con códigos HTTP apropiados y mensajes descriptivos, y agregué sanitización de inputs para prevenir inyecciones SQL y XSS.

Integración Frontend-Backend Inconsistente

Comunicación desalineada entre el equipo de frontend y backend resultaba en cambios de contratos de API no documentados y problemas de integración frecuentes.

Solución Implementada

Establecí documentación automática de APIs, definí contratos claros para cada endpoint, implementé versionado de API, y coordiné reuniones regulares de sincronización entre equipos. Esto redujo drásticamente los bugs de integración y aceleró el desarrollo.

Highlights Técnicos

Arquitectura en Capas

Implementación de arquitectura limpia separando responsabilidades:

  • Controllers: Manejo de requests/responses HTTP
  • Services: Lógica de negocio y orquestación
  • Repositories: Acceso a datos y queries
  • Middlewares: Validación, autenticación y manejo de errores
  • DTOs: Data Transfer Objects para contratos de API claros

Optimización de Base de Datos

  • Diseño de índices compuestos para queries frecuentes
  • Implementación de eager loading para reducir N+1 queries
  • Uso de transacciones para operaciones críticas
  • Migraciones versionadas para control de cambios en schema
  • Query optimization mediante EXPLAIN ANALYZE

Seguridad y Validación

  • Validación de inputs con class-validator y class-transformer
  • Sanitización de datos para prevenir SQL injection y XSS
  • Implementación de rate limiting para prevenir ataques DDoS
  • Manejo seguro de credenciales y secrets con variables de entorno
  • CORS configurado apropiadamente para entornos de producción
