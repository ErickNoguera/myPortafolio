El Proyecto

Auto-volt.cl es una plataforma real de electromovilidad en Chile que centraliza toda la oferta de vehículos eléctricos e híbridos del país. Me uní al proyecto en Junio 2025 como Junior Backend Developer en modalidad voluntaria para adquirir experiencia en un producto real en producción.

Trabajo directamente con Nicolás Núñez, socio fundador y líder técnico con más de 12 años de experiencia, quien supervisa mi trabajo mediante revisiones de código en GitHub. Aunque es voluntario (sin remuneración), esta oportunidad me está permitiendo aprender en un entorno profesional real, trabajar con metodología Scrum, y contribuir a un MVP funcional que arrancó desde cero.

El desafío principal del proyecto era automatizar la extracción de información de más de 89 automotoras chilenas para mantener actualizado el catálogo de vehículos. Inicialmente trabajábamos con Selenium, pero propuse crear un microservicio independiente con Node.js + TypeScript + Stagehand (framework con IA), lo cual mejoró la certeza de datos extraídos del 60% al 90%.

El Problema

El equipo venía trabajando con Selenium en Python para scraping de las 89+ automotoras. Sin embargo, la certeza de datos extraídos era solo del 60% debido a cambios constantes en las estructuras HTML de los sitios web, lo que generaba datos inconsistentes y mucho trabajo manual de corrección.

Mi Propuesta

Investigué y descubrí Stagehand, un framework de web scraping potenciado con IA que se adapta automáticamente a cambios en las páginas. Propuse crear un microservicio independiente con Node.js + TypeScript que funcionara en paralelo al código principal de Django.

El Desafío de Convencer al Equipo

No fue fácil. Nicolás es experto en Python/Django, pero no conocía TypeScript ni Node.js. Proponer introducir un stack completamente diferente significaba que yo asumiría toda la responsabilidad del microservicio. Me costó convencerlos, pero logré demostrar que valía la pena:

  • Presenté investigación sobre Stagehand y sus ventajas con IA
  • Expliqué por qué Node.js era mejor para este caso específico (async, npm ecosystem)
  • Me comprometí a ser el único responsable de su desarrollo y mantenimiento
  • Propuse dockerizarlo para que fuera completamente independiente

Desarrollo del Microservicio

  • 2 semanas: Montar la infraestructura base (Node.js, TypeScript, Stagehand, Docker)
  • 2 semanas más: Poner a funcionar y extraer información correctamente
  • Total: 1 mes desde idea hasta producción funcional
  • Trabajé completamente solo en esta parte del sistema

Resultados Medibles

  • 60% → 90%: Mejora en certeza de datos extraídos
  • 89+ automotoras: Scraped de forma automatizada
  • Microservicio independiente: Dockerizado y desacoplado
  • Menor mantenimiento: IA se adapta a cambios en sitios web
  • Producción: Funcionando actualmente en auto-volt.cl

Detalles Técnicos del Microservicio

Stagehand con IA

  • Framework de scraping que usa IA para entender estructura de páginas
  • Se adapta automáticamente a cambios en HTML/CSS sin romper scrapers
  • Reduce mantenimiento: no necesito reescribir selectores cada vez que cambian sitios
  • Mejora extracción de datos no estructurados usando visión artificial
  • Maneja JavaScript dinámico y SPAs sin problemas

Arquitectura Dockerizada

  • Dockerfile optimizado para Node.js + TypeScript
  • Multi-stage build para reducir tamaño de imagen final
  • Docker Compose para orquestar microservicio + DB + servicios auxiliares
  • Variables de entorno para configuración por ambiente (dev/prod)
  • Volúmenes para persistencia de datos scraped

Mejora de Certeza: 60% → 90%

  • Antes (Selenium): Selectores CSS frágiles que rompían con cambios mínimos
  • Después (Stagehand IA): IA entiende contexto y encuentra datos automáticamente
  • Validación cruzada de datos extraídos con múltiples fuentes
  • Sistema de confianza: marca datos con probabilidad de certeza
  • Logs detallados para debugging cuando hay inconsistencias

Integración con Backend Django

  • API REST para comunicación entre microservicio y Django
  • Cola de tareas para scraping asíncrono (no bloquea requests)
  • Webhooks para notificar a Django cuando hay datos nuevos
  • Sistema de retry con exponential backoff para sitios lentos
  • Cache de resultados para evitar re-scraping innecesario
