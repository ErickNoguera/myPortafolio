Sobre el Proyecto

Erick Films fue mi primer proyecto Full Stack completo, desarrollado durante el Módulo 2 del bootcamp de Henry. Por primera vez conecté un frontend propio con un backend que yo mismo construí, manejé una base de datos en la nube, y estructuré un proyecto profesional con arquitectura separada cliente-servidor.

El desafío consistía en crear un catálogo de películas funcional donde los usuarios pudieran visualizar películas desde la base de datos y agregar nuevas mediante un formulario con validaciones. Aunque seguimos la supervisión del instructor, cada estudiante implementó su propia solución de forma individual, cumpliendo requisitos técnicos específicos.

Los requisitos obligatorios incluían: formulario funcional con validaciones, base de datos no relacional (MongoDB), separación clara entre frontend y backend, consumo de API desde el cliente, y código modular bien estructurado. Este proyecto me enseñó los fundamentos de arquitectura Full Stack.

Mi Primer Full Stack

Contexto: Módulo 2 del Bootcamp Full Stack de Henry
Modalidad: Proyecto individual con supervisión del instructor
Requisitos: Formulario, MongoDB, separación frontend/backend, código modular
Logro: Primera aplicación completa conectando todas las piezas del stack

¿Qué Construí?

Frontend Modular con Webpack

Construí el frontend con JavaScript vanilla puro, sin frameworks. Usé Webpack para empaquetar múltiples archivos JS en un bundle optimizado, configuré scripts de desarrollo y producción, y organicé el código en módulos separados por responsabilidad.

API REST con Express

Servidor Node.js + Express con arquitectura profesional: carpetas separadas para config/, controllers/, middleware/, models/ y routes/. Implementé endpoints CRUD completos, manejo de errores global y configuración de CORS.

MongoDB en la Nube

Primera vez usando MongoDB Atlas. Configuré la conexión con variables de entorno, creé esquemas con Mongoose, definí validaciones en el modelo, y manejé operaciones CRUD desde el backend hacia la base de datos.

Catálogo Visual de Películas

Grid de películas mostrando posters, efecto hover interactivo que revela información adicional, click en tarjeta que abre modal con detalles completos (título, año, director, duración, género, rating), diseño responsivo y atractivo.

Formulario con Validaciones

Formulario completo para agregar películas con campos: título, año, director, duración, género (selector), calificación y URL del poster. Validaciones tanto en cliente (JavaScript) como en servidor (Mongoose), feedback visual de éxito/error.

Comunicación con Axios

Consumo de la API REST desde el frontend usando Axios. Aprendí a manejar peticiones HTTP asíncronas, procesar respuestas JSON, manejar errores de red, y actualizar la interfaz dinámicamente cuando se agregan películas.

Stack Tecnológico

Frontend

HTML5 CSS3 JavaScript Vanilla Webpack Axios Live Server

Frontend construido con JavaScript vanilla modularizado. Configuré Webpack desde cero para bundling, minificación y compilación. Usé Axios para consumo de API y organicé el código en módulos separados sin depender de frameworks - JavaScript puro y bien estructurado.

Backend

Node.js Express.js Mongoose CORS Dotenv

API REST con Express siguiendo arquitectura modular profesional: config/ para configuraciones, controllers/ para lógica de negocio, middleware/ para funciones intermedias, models/ para esquemas Mongoose, y routes/ para endpoints. Variables de entorno con dotenv y CORS configurado.

Base de Datos

MongoDB MongoDB Atlas Mongoose ODM

Primera vez usando MongoDB en la nube con Atlas. Aprendí a construir connection strings dinámicos desde variables de entorno, definir esquemas con validaciones, manejar conexiones con pool de conexiones, y realizar operaciones CRUD en una base de datos NoSQL.

Herramientas de Desarrollo

Git/GitHub npm Scripts Webpack Config Environment Variables

Configuré scripts de npm para desarrollo (npm run dev) y producción (npm run build), webpack.config.js para compilación y empaquetado, y variables de entorno para gestionar credenciales de MongoDB de forma segura sin exponerlas en el código.

Desafíos Técnicos y Aprendizajes

Configurar Webpack por Primera Vez

Nunca había configurado un bundler. No entendía entry points, output paths, loaders o cómo Webpack convierte múltiples archivos JS en un solo bundle optimizado.

Lo que Aprendí

Leí la documentación de Webpack, experimenté con diferentes configuraciones en webpack.config.js, y entendí el flujo de compilación. Aprendí a configurar entry/output, usar loaders para CSS, y crear scripts separados para desarrollo y producción. Ahora comprendo por qué los bundlers son importantes en proyectos modernos.

Conectar Frontend con Backend

Era mi primera vez conectando un cliente con un servidor propio. No sabía cómo hacer que el frontend corriendo en localhost:8080 se comunicara con el backend en localhost:3000 sin errores de CORS.

Lo que Aprendí

Investigué sobre CORS y por qué los navegadores bloquean peticiones cross-origin. Configuré el middleware de CORS en Express, entendí la diferencia entre desarrollo y producción, y aprendí a usar Axios para hacer peticiones HTTP de forma limpia con promesas.

Manejar Variables de Entorno

No quería exponer las credenciales de MongoDB en el código, pero no sabía cómo usar variables de entorno ni cómo construir el connection string dinámicamente.

Lo que Aprendí

Implementé dotenv para cargar variables desde .env, agregué .env al .gitignore para seguridad, y construí el connection string de MongoDB dinámicamente usando template literals con process.env.DB_USER, DB_PASSWORD, etc. Entendí la importancia de no versionar credenciales.

Validaciones en Cliente y Servidor

Sabía que debía validar datos, pero no estaba claro dónde hacerlo: ¿solo en el cliente? ¿solo en el servidor? ¿En ambos lados?

Lo que Aprendí

Implementé validaciones en ambos lados: JavaScript en el cliente para feedback inmediato al usuario, y validaciones en el esquema de Mongoose en el servidor como última línea de defensa. Aprendí que nunca se debe confiar solo en validaciones del cliente porque pueden ser bypasseadas.

Aprendizajes Clave

Conceptos Fundamentales

  • Arquitectura Cliente-Servidor: Separación clara entre frontend y backend, cada uno con su propia responsabilidad
  • API REST: Cómo diseñar endpoints, métodos HTTP (GET, POST, PUT, DELETE), y estructura de respuestas
  • Bases de Datos NoSQL: Diferencias con SQL, esquemas flexibles, y cuándo usar MongoDB
  • Build Tools: Por qué existen bundlers como Webpack y cómo mejoran el desarrollo
  • Seguridad Básica: No exponer credenciales, validar siempre en servidor, configurar CORS correctamente

Habilidades Técnicas Adquiridas

  • Configurar Webpack desde cero para proyectos JavaScript vanilla
  • Estructurar proyectos backend con arquitectura modular profesional
  • Conectar aplicaciones a MongoDB Atlas en la nube
  • Manejar variables de entorno de forma segura
  • Consumir APIs con Axios y manejar operaciones asíncronas
  • Implementar validaciones robustas en cliente y servidor

Por Qué Este Proyecto Fue Importante

Este fue el momento en que todo "hizo click". Por primera vez vi cómo todas las piezas del stack se conectan: el usuario interactúa con HTML/CSS/JS en el navegador, el JavaScript hace peticiones HTTP a un servidor Express, el servidor consulta MongoDB, y la respuesta viaja de vuelta al cliente para actualizar la UI.

Aunque fue un proyecto académico, sentó las bases fundamentales que apliqué después en proyectos más complejos como el E-commerce con NestJS y Si, Voy. Aprendí que la arquitectura Full Stack no es complicada si entiendes bien cada capa y cómo se comunican entre sí.

Detalles de Implementación

Estructura del Proyecto

  • front/: Cliente con HTML, CSS, JS y Webpack config
  • front/src/: Código fuente JavaScript modularizado
  • front/dist/: Bundle compilado por Webpack
  • back/config/: Configuración de conexión a MongoDB
  • back/controllers/: Lógica de negocio de cada endpoint
  • back/middleware/: Funciones intermedias (validaciones, errores)
  • back/models/: Esquemas de Mongoose para películas
  • back/routes/: Definición de endpoints de la API

Scripts Configurados

  • Frontend - npm run dev: Compilación en modo desarrollo con Webpack
  • Frontend - npm run build: Compilación optimizada para producción
  • Frontend - npm start: Build + servidor con Live Server
  • Backend - npm start: Inicia servidor Express en puerto configurado
  • Flujo: Backend debe estar corriendo antes de levantar el frontend

Conexión a MongoDB

  • Connection string construido dinámicamente desde variables .env
  • Formato: mongodb+srv://user:password@host/dbname?options
  • Pool de conexiones configurado (maxPoolSize: 10)
  • Logs de conexión para debugging (connected/error/disconnected)
  • Manejo de errores de conexión con try/catch

Características del Frontend

  • JavaScript modularizado sin frameworks (vanilla JS puro)
  • Webpack empaqueta múltiples archivos en un solo bundle
  • Axios para todas las peticiones HTTP a la API
  • Manipulación del DOM para renderizar películas dinámicamente
  • Event listeners para formulario, hover effects y modales
  • Validaciones de campos antes de enviar al servidor
